The Classic Mushroom Dataset

Preprocessing

So the first thing I did was look over the data in a text editor. There was nothing that jumped out at me mostly because it was information overload. 23 elements is a lot to look at when it's all represented as single letters. I then looked over the meta-data sheet. The information there was helpful to know. The main thing I noted was that one of the characteristics had missing values. About a fourth of the data set was missing a value for stalk-root. I decided that the best rout for this was to simply remove that element from analysis. There are 21 other elements that should contain enough information and with that much data missing I though it allowed for enough uncertainty. I also didn't see the data set lending it's self to stick in a average or default value.

Summary Statistics

I looked over the basic summary statistics. Because all the values were nominal there wasn't really averages or any good number statistics to look at. However you can simply look at modes of different characteristics and see what is abundant and how abundant it is.

Classification

I used WEKA to analyze this data. I used a tree classifier using Simple Cart from the options and it created a Cart of 7 leaf nodes. It was 99.9 % effective at distinguishing poisonous from not poisonous. It turned out to be very similar to what was given in the meta data which was kind of boring however it was validating the information given. On trying to use other methods of classification I often crashed WEKA because it ran out of memory because of the size of the dataset and the limits of my computer.

Anomalies

I didn't look into the anomalies because WEKA doesn't really support it and my computer is bitching about Knime. However I assume there are anomalies because with a real world dataset of over 8000 items there are bound to be outliers.

Association Rules

Because I created a simple and through tree the branches can be rewrote into an effective set of rules to categorize the data.
